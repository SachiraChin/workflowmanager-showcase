# Issue: WebUI Debug Mode for Schema Rendering

## Summary

Add a debug mode toggle for WebUI that shows schema rendering diagnostics inline, helping developers identify issues with role types, slot assignments, and display modes.

## Background

During schema renderer refactoring, we identified that role type warnings (e.g., `card-title` used outside card context) are currently logged to console. This makes debugging difficult as developers need to check both UI and console.

## Proposed Feature

### Debug Mode ON

- Show role type warnings inline below the affected field
- Show slot assignment indicators (e.g., small "title" badge on card-title fields)
- Show display mode indicators (visible/hidden/passthrough)
- Highlight fields that are being dropped or not rendering as expected

### Debug Mode OFF (Default)

- Render normally
- No visual indicators
- Minimal or no console warnings

## UI Suggestions

```
┌─────────────────────────────────────────┐
│ [Card]                                  │
│ ┌─────────────────────────────────────┐ │
│ │ Item Title           [slot: title]  │ │
│ │ ⚠️ card-subtitle outside card       │ │
│ │ Description text     [slot: body]   │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

## Implementation Notes

- Toggle could be in browser localStorage or URL param (`?debug=1`)
- Use React Context to propagate debug mode state
- Add conditional rendering for debug badges/warnings
- Keep debug components tree-shakeable for production

## Priority

**Low** - Developer tooling enhancement, not blocking any features.
